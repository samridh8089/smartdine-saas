## How to Restore

### 1. Environment Setup

```bash
# Clone or unzip the backup
cd SmartDine-Backup-2026-06-29_11-38

# Install dependencies
npm install

# Create environment file
copy .env.example .env.local
# Then edit .env.local and fill in:
# NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
# NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
```

### 2. Database Restore

Go to your Supabase project → SQL Editor and run **DATABASE-BACKUP.sql** or apply each migration file from `supabase/migrations/` in order:

1. `20260624000000_schema.sql`
2. `20260625000000_schema_updates.sql`
3. `20260626000000_realtime_rls_fixes.sql`
4. `20260626000001_restaurant_alert_workflow.sql`
5. `20260627000000_multi_batch_orders.sql`
6. `20260627000001_order_batches_rls_fix.sql`
7. `20260627000002_billing_trigger.sql`
8. `20260627000003_order_batches_public_select.sql`
9. `20260628000000_activity_log_and_cancellation.sql`
10. `20260628000001_customer_upi_payment.sql`
11. `20260628000002_takeaway_fields.sql`

After applying all migrations, run:
```sql
NOTIFY pgrst, 'reload schema';
```

### 3. Run Locally

```bash
npm run dev
# Open: http://localhost:3000
```

### 4. Deploy to Vercel

```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel --prod
# Set env vars in Vercel dashboard:
# NEXT_PUBLIC_SUPABASE_URL
# NEXT_PUBLIC_SUPABASE_ANON_KEY
```
